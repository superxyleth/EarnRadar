# EarnRadar 开发进度

更新日期：2026-06-18

## 项目概览

EarnRadar 是一个静态前端活动雷达页面，用于集中展示 Binance / OKX 的理财、Alpha、链上赚币、Boost 与交易任务信息。项目当前不依赖构建工具，核心由 `index.html`、`styles.css`、`data.js` 三个文件组成，直接通过浏览器打开即可运行。

## 当前状态

- 页面框架已完成：包含顶部品牌栏、作者入口、左侧交易所/模块导航、主内容列表、统计概览和搜索框。
- 数据渲染已完成：`data.js` 通过 `window.financeActivities` 提供活动数组，`index.html` 内联脚本负责到期下架、过滤、排序、统计和卡片渲染。
- 交互能力已完成：支持按交易所切换、按模块切换、关键词搜索、活动规则折叠展开、参与入口和官方公告跳转。
- 视觉样式已完成：采用深色底色与金色强调色，包含卡片、标签、按钮、摘要区域、滚动条和移动端响应式布局。
- 静态素材已准备：包含作者头像、桌面端/移动端页面截图和若干 Binance 活动原始 JSON 数据文件。

## 待完善事项

- 数据更新仍是手动维护：后续可以把原始 JSON 到 `data.js` 的整理流程脚本化，减少录入错误。
- 活动状态未自动计算：目前已经按 `endAt` 自动下架过期活动，但状态文案仍依赖数据字段，建议根据当前时间、`startAt` 和 `endAt` 自动派生“未开始 / 进行中 / 已结束”。
- 时间字段缺少统一解析层：建议集中封装时间解析和时区展示，避免未来跨时区活动出现排序或状态偏差。
- 搜索范围可以扩展：当前未覆盖规则摘要、来源说明等字段，后续可纳入全文检索。
- 缺少数据校验：建议增加活动字段完整性检查，例如 `id` 唯一性、URL 合法性、必要字段是否为空。
- 缺少自动化验证：当前项目没有测试或 lint 配置，后续可加入轻量脚本检查 HTML、CSS 和数据文件可解析性。
- 历史活动归档策略未定义：随着活动数量增加，需要决定已结束活动是隐藏、归档还是保留在列表中。

## 建议下一步

1. 增加一个数据校验脚本，检查 `data.js` 的活动数量、必填字段、重复 ID、URL 和日期格式。
2. 将活动状态改为自动计算，并在页面增加“全部 / 进行中 / 已结束”的状态筛选。
3. 把 `index.html` 内联脚本拆分为独立 JS 文件，便于维护和后续测试。
4. 建立活动数据更新流程：从公告 JSON 提取字段，生成或合并到 `data.js`。
5. 更新截图素材，保留一张桌面端和一张移动端最新效果图即可，避免根目录素材越来越散。
